## test情况1 （没有用Lisa）
### 如何避免显存不足的情况
为避免训练或验证过程中出现显存不足，我先后尝试了多种方法。首先，在参数设置上，我通过减小 batch_size、增大 grad_accumulation_steps 的方式，降低了单步前向和反向传播时的显存占用，同时尽量保持等效 batch 大小不至于过小。其次，在模型计算过程中，我启用了 gradient checkpointing，用额外的计算时间换取更低的中间激活存储开销，这对缓解显存压力起到了比较明显的作用。

另外，我也尝试过从输入规模入手，例如调整图像尺寸、控制 model_max_length 等，以减少视觉 token 和文本 token 带来的显存消耗。不过这类方法虽然能降低显存占用，但有时也会引入尺寸不匹配等新的问题，因此需要和代码实现保持一致。除此之外，我还根据服务器上的 GPU 使用情况调整运行策略，尽量选择空闲显存更多的卡运行任务，并避开存在故障的 GPU，从而减少因显存不足或设备异常导致的中断。

在实验方案上，我也采用了“先降低任务规模再逐步推进”的思路。例如在模型规模上优先选择 7B 而不是 13B，在数据上优先测试较小范围或单一任务设置，而不是一开始就运行完整配置。这样做可以先保证程序能够跑通，再逐步扩展到更完整的实验。总体来说，我为了避免显存不足，主要采取了减小 batch、大梯度累积、启用梯度检查点、适当压缩输入规模、合理选择 GPU，以及优先使用较小模型和较轻配置等方法。
